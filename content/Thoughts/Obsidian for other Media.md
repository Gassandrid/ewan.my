Not sure how I would want to go about this, but it itching me in a few ways.

## Paper Management

This one is already mainly feature complete, but I would like a way to have all my zotero papers get an equivelant file in my obsidian vault. Not super important, but saves me the hastle of having to import with two firefox extensions

>[!Success] Completed [[12-16-2025]]
> this was actually really easy to do! The zotero integration plugin had a method for this I was not aware of via note templates, this still nicely integrates with zotero by importing annotations, while bringing all of the features I wanted to obsidian ( even better author name parsing! )

## Image Management

This idea stems from the work of location based notes. I quite like having them, but I feel as though managing images through this interface would be nice. For example, if I could search image metadata via bases this would allow me to have an inline base for each location note with its associated images. 

Perhaps a better implementation would be to integrate with [[Immich]], as that is already the best service for the job when it comes to immage and video management. Unsure how querying would work though. But this is better, as apon deeper thought it really doesnt make sense for obsidian to be a media management engine. Too much stuff for random pasted images off the internet, not the place for all personal family photos. **Scrap Book**, not **Photo Gallery**.

Will write about this in [[On Curating Images]].
